# Контрольная работа 2

## Описание проекта

Необходимо разработать веб-приложение для создания и управления заметками с категориями с использованием **Class-Based Views (CBV)**.

Пользователь должен иметь возможность создавать заметки, организовывать их по категориям, просматривать, редактировать и удалять.

---

## Требования к моделям

### Модель `Category`
- `name` — название категории
- `user` — владелец категории (связь с моделью `User`)
- `created_at` — дата создания

### Модель `Note`
- `title` — заголовок заметки
- `content` — текст заметки
- `category` — категория заметки (ForeignKey к `Category`, может быть пустой)
- `user` — владелец заметки (связь с моделью `User`)
- `is_pinned` — закреплена ли заметка (по умолчанию `False`)
- `created_at` — дата создания
- `updated_at` — дата последнего обновления

> **Важно:** Заметки и категории должны принадлежать только текущему пользователю.

---

## Обязательный функционал

- Регистрация и авторизация пользователей
- Создание, просмотр, редактирование и удаление заметок
- Создание категорий
- Просмотр списка всех заметок пользователя
- Фильтрация заметок по категории
- Просмотр детальной информации по одной заметке
- Возможность закреплять заметки (`is_pinned`)
- При создании и редактировании заметки пользователь должен иметь возможность выбрать категорию
- В списке заметок закреплённые заметки должны отображаться первыми (order_by)
